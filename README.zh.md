# 智能个人助手

一套用 markdown 文件构成的 AI 生活管理框架。把这些文件给任何 AI——ChatGPT、Claude、豆包、Gemini、本地模型——它就能变成你的个人生活助理。

框架是产品。守护进程是可选的基础设施。

---

## 概览

智能个人助手是一组 markdown 文件，教任何 AI 如何帮你管理多个项目、追踪任务、规划每天、保持井然有序。两种使用方式：

1. **纯框架** — 把文件给任何 AI 直接聊。不需要代码，不需要安装。
2. **守护进程模式** — 一个轻量 Python 脚本添加主动功能：早间日报、定时签到、提醒和 Slack 即时响应。

---

## 框架

框架是一组 markdown 文件，定义了如何组织工作、确定优先级、管理日程。任何读取这些文件的 AI 都能作为你的助手。

### 核心文件

| 文件 | 用途 |
|------|------|
| `START HERE.md` | 入口——告诉 AI 该做什么、如何行动 |
| `Cognitive Levels.md` | L1/L2/L3 系统——按"需要你多少脑力"分类任务 |
| `Priority Framework.md` | 艾森豪威尔矩阵——按重要性×紧急性分类 |
| `Cowork Agent Playbook.md` | 日报、签到、总结的模板 |
| `Getting Started.md` | 入门引导——AI 面试你来设置系统 |

### 用户文件（入门时创建）

| 文件 | 用途 |
|------|------|
| `Workstreams.md` | 你的项目、任务、优先级和上下文 |
| `Weekly Goals.md` | 本周各项目的目标 |
| `Daily Scaffolding.md` | 按容量映射的日程——什么时候建议什么 |
| `Preferences.md` | 长期记忆——你给机器人的行为规则 |
| `Automations.md` | 定时事件和提醒（可读视图） |

### 核心理念

**认知层级** — 不是所有任务都需要同样的脑力：
- **L1** — 深度专注（设计、debug）。不能做别的。
- **L2** — 轻度参与（AI 在做，你在看）。一次一个。
- **L3** — 自主运行（AI 独立完成）。无限并行。
- **规则：** 绝不同时两个 L2。

**艾森豪威尔矩阵** — 每个新事项自动分类：
- Q1（重要+紧急）→ 立刻做
- Q2（重要+不紧急）→ 保护时间做这个
- Q3（不重要+紧急）→ 委派或快速处理
- Q4（不重要+不紧急）→ 放弃

**每日节奏** — 按容量映射你的一天。AI 在对的时间建议对的事。

**周循环** — 周日计划 → 每日日报 → 周五回顾。

## 插件

插件用领域特定的功能扩展助手。每个插件是一个带 `playbook.md` 的文件夹，教 AI 新的能力。

### 内置插件

| 插件 | 功能 |
|------|------|
| **grocery** | 按商店类型分类的购物清单（Costco、普通超市、中国超市）。自动分类物品。 |
| **travel** | 旅行打包，有缩放规则。根据天数、天气、出行人员生成打包清单。 |

### 插件如何工作

1. 告诉机器人："安装购物清单插件"
2. 机器人读取 playbook，在 `Preferences.md` 添加路由规则
3. 以后匹配的请求（"加牛奶到购物清单"）自动路由到该插件
4. 插件数据在你的笔记文件夹：`plugins/grocery/my_grocery.md`

### 创建自定义插件

**最简单的方式：** 直接告诉你的 AI 你想追踪什么。

```
你：我想追踪我的健身和运动目标
机器人：好的，我帮你创建一个健身插件。它可以：
       - 记录运动（类型、时长、备注）
       - 设定每周健身目标
       - 显示进度总结
       已添加到你的偏好设置并创建了 playbook。
       试试说"记录今天跑步30分钟"。
```

AI 会创建 `playbook.md`、在你的 `Preferences.md` 添加路由规则、创建数据文件——全部通过对话完成。不需要手动编辑文件。

**手动创建**（如果你喜欢的话）：在 `plugins/my_plugin/` 创建 `playbook.md`，描述安装规则、行为规则和数据文件格式。

插件支持中文（`plugins/zh/`）和英文（`plugins/en/`）。

## 守护进程

守护进程是一个轻量 Python 脚本，添加框架本身无法提供的主动功能：定时日报、即时消息处理和自动提醒。

### 如何接收消息

**Slack Socket Mode**（首选）— 即时事件驱动。你发 Slack 消息，守护进程立即处理。

**轮询**（后备）— 每 60 秒检查一次新消息。Socket Mode 不可用时使用。

### 如何调用 AI

守护进程只在需要智能时调用 `claude -p`（Claude CLI）。通过 stdin 传递提示，接收纯文本返回。每次调用独立——无会话、无状态。

**Token 用量：** 全功能约 15-20K/天。包含在 Claude Pro（$20/月）中。

### 零权限设计

AI 从不直接访问文件或 API。守护进程处理所有 I/O：

```
守护进程读取文件 → 构建提示 → 通过 stdin 发送给 AI
AI 返回 JSON 文本 → 守护进程解析 → 发到 Slack + 写入文件
```

无工具权限、无 MCP、无文件系统访问。AI 是纯文本输入、纯文本输出。

## 开始使用

### 方式一：纯框架（任何 AI，无需安装）

**第 1 步：下载框架**
```bash
git clone https://github.com/siyigu1/smart-personal-assistant.git
```
复制 `framework/zh/` 文件夹到方便的地方。

**第 2 步：分享给你的 AI**

打开你常用的 AI，粘贴这段提示：
```
我在使用一个个人生活管理框架。请读取附件中以"START HERE.md"开头的文件，
然后帮我管理我的一天。
```

上传 framework/zh/ 里的所有 `.md` 文件。

**第 3 步：入门设置**

AI 会用对话方式了解你的日程、项目和偏好（约 15 分钟）。它创建你的个人文件：Workstreams.md、Daily Scaffolding.md、Weekly Goals.md 和 Preferences.md。

**第 4 步：日常使用**

每次开新对话时分享你的文件。可以问：
- "我该做什么？"
- "加牛奶到购物清单"（安装了购物清单插件的话）
- "帮我分类这个新任务"
- "给我跑一个早间日报"

> 没有守护进程的话，AI 只在你发消息时响应。想要主动推送（早间日报、自动提醒），用守护进程。

---

### 方式二：守护进程模式（主动功能）

**前提条件：**
- Python 3.10+
- Claude CLI（`claude`）+ Pro/Max 订阅
- Slack 工作区

**第 1 步：克隆并运行设置**
```bash
git clone https://github.com/siyigu1/smart-personal-assistant.git
cd smart-personal-assistant
./setup.sh
```

设置向导引导你完成语言选择、名字、Slack 应用创建、文件夹位置和家庭扩展。

**第 2 步：启动守护进程**
```bash
./run.sh
```

**第 3 步：入门设置**

AI 在 Slack 中联系你，用对话方式了解你的日程和项目。回答后它创建所有文件并设置自动化任务。

**设置完成后：**
```bash
# 启动 / 停止
./run.sh                 # 启动守护进程（前台）
launchctl stop com.mission-control.daemon   # 停止（macOS 后台）
launchctl start com.mission-control.daemon  # 启动（macOS 后台）
systemctl --user stop mission-control       # 停止（Linux 后台）
systemctl --user start mission-control      # 启动（Linux 后台）

# 监控
./status.sh              # 查看守护进程状态
./status.sh activity     # 最近活动日志
./status.sh errors       # 最近错误
./run.sh --once          # 测试一个周期
```

## 系统深入

### 架构

```
┌──────────────────────────────────────────────────────┐
│              框架（独立，任何 AI）                      │
│                                                       │
│  用户笔记文件夹：                                      │
│  ├── reference/     基础方法论文件                     │
│  ├── Workstreams.md  用户的项目和任务                   │
│  ├── Preferences.md  长期记忆                          │
│  └── plugins/        插件数据文件                      │
└───────────────────────┬──────────────────────────────┘
                        │ 守护进程读/写
                        ▼
┌──────────────────────────────────────────────────────┐
│              守护进程（确定性）                         │
│                                                       │
│  Socket Mode → 消息即时处理 → 调用 AI                  │
│  定时器 → 自动化任务 + 维护（凌晨1-4点）                │
│  data/{user}/ → 内部数据（不在 Obsidian）              │
└───────────────────────┬──────────────────────────────┘
                        │ 需要智能时才调用
                        ▼
┌──────────────────────────────────────────────────────┐
│              AI（纯文本输入/输出）                      │
│                                                       │
│  返回 JSON：消息、文件更新、自动化操作、记忆             │
└──────────────────────────────────────────────────────┘
```

### 记忆系统

| 层级 | 位置 | 有效期 | 用途 |
|------|------|--------|------|
| 对话 | `data/{user}/conversation-state.json` | 2小时 | 多轮对话日志 |
| 短期 | `data/{user}/short-term-memory.json` | 7天 | 跨消息上下文 |
| 长期 | `Preferences.md`（Obsidian 中） | 永久 | 用户规则 |

每 7 天，守护进程整理：重要的短期记忆移到 Preferences.md，过期条目清除。

### 自动维护（凌晨 1-4 点）

| 任务 | 频率 | 需要 AI？ |
|------|------|----------|
| 记忆整理 | 7天 | 是 |
| 文件整理 | 7天 | 是 |
| 系统更新检查 | 每天 | 否 |
| AI 工具更新检查 | 每天 | 否 |

结果写入 System Notices.md，AI 在早间日报中自然提及。
