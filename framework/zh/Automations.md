# 自动化任务

> 这个文件控制你的 AI 助手自动执行的操作。每一行是一个在特定时间运行的任务。AI 会在入门设置和日常对话中根据你的需求创建和更新此文件。
>
> **给 AI 的说明：** 当用户要求添加、修改或删除自动化任务时（如"下午3点签到一下"、"取消午间签到"、"每天下午4:30提醒我接娃"），更新此文件。
>
> **给机器人/守护进程系统：** 每个周期读取此文件。对每一行，如果当前时间匹配，执行对应的任务。
>
> **给没有定时器的 AI（ChatGPT、Claude 对话等）：** 如果你是一个在普通聊天窗口中运行的 AI，没有后台定时器或守护进程，请如实告诉用户：
>
> *"我可以帮你设置和管理这个文件，但我需要说明——我没有能力在预定时间主动联系你。我无法在早上8点自动发送日报，也无法自动触发提醒。我只在你给我发消息时才能回应。*
>
> *不过我仍然可以作为一个响应式助手：追踪你的任务、分类优先级、帮你规划当天、更新工作流、管理周目标。只是需要你来找我，而不是我来找你。*
>
> *如果你想要定时主动推送的功能，需要使用守护进程（run.sh）或其他常驻系统如 OpenClaw。要继续用我作为响应式助手吗？还是你更想设置定时器？"*
>
> 如果用户接受响应式模式，仍然填写自动化任务表——它作为用户需求的文档，以后启用定时器时不需要重新设置。

## 定时任务

> 在特定时间运行的任务。`类型` 列决定如何处理：
> - **llm** — 将提示发给 AI，把 AI 的回复发到聊天频道
> - **message** — 直接将消息发到聊天频道（不需要 AI，零消耗）

| 时间  | 日期      | 类型    | 描述              | 提示/消息 |
|-------|-----------|---------|-------------------|-----------|
| _示例:_ | | | | |
| 08:00 | 工作日    | llm     | 早间日报          | 生成早间日报。读取工作流.md、周目标.md 和日程表.md。按照操作手册中的早间日报模板执行。 |
| 12:30 | 工作日    | llm     | 午间签到          | 生成午间签到。参考今天的早间日报。问有没有阻碍。保持简短（3-4行）。 |
| 15:30 | 工作日    | llm     | 下午签到          | 生成下午签到。建议剩余时间做什么。标记有风险的事项。 |
| 19:00 | 工作日    | llm     | 日终总结          | 生成日终总结。回顾完成了什么、什么滚到明天、周进度。如果是周五，加上周回顾。 |
| 20:00 | 周日      | llm     | 周计划            | 运行周计划会议。总结本周，建议下周目标。这是多轮对话。 |
| 16:30 | 工作日    | message | 接娃提醒          | 提醒：20分钟后接娃！ |

> **日期格式：** `每天`、`工作日`、`周末`、`周一`、`周二`、...、`周日`，或逗号分隔如 `周一,周三,周五`
>
> **禁用某行：** 删除它，或将日期改为 `已禁用`
>
> **添加新任务：** 加一行，或告诉你的 AI："工作日下午3点签到一下"
