# 优先级框架

> **艾森豪威尔矩阵应用于生活管理系统。** 每当有新事项出现——任务、问题、想法、任何可能变成行动项的东西——都主动分类。不要等被问到才分类。

---

## 四个象限

### Q1：重要 + 紧急 → 最高优先级
既重要又有迫在眉睫的截止日期。立即安排到下一个可用的深度工作时段。

**信号：** 几天内有硬性截止日期、阻塞其他工作、延迟会有重大后果、直接推进高优先级工作流目标。

**行动：** 添加到周目标，标记为 P1。建议今天或明天的具体时间段。

---

### Q2：重要 + 不紧急 → 高优先级（保护这个）
重要但没有即时压力。这里才是最有价值的工作所在——战略思考、规划、技能提升、关系建设。危险在于 Q1 和 Q3 会把它挤掉。

**信号：** 能显著推进某个工作流、本周没有硬性截止日期但有未来里程碑、能建立能力或预防未来的问题、不做会后悔。

**行动：** 添加到周目标，标记为 P1 或 P2。为它保护时间——建议安排到深度工作时段，趁它还没变紧急。如果已经搁置超过一周，标记提醒。

---

### Q3：不重要 + 紧急 → 委派或快速处理
有时间压力但实际影响小。感觉紧急但不推动进展。目标是花最少精力处理。

**信号：** 有人在等、有截止日期，但不推进核心工作流或目标。行政任务、别人的请求、有期限的例行维护。

**行动——按顺序尝试：**
1. **委派给 agent（L3/L2）：** 如果能自主或轻度监督处理，建议作为 agent 任务。
2. **委派给其他人：** 如果有人能处理（伴侣、合作者、服务提供商），建议委派。
3. **快速处理：** 如果确实 15 分钟内能搞定，建议塞到过渡时间段（会议间隙、做饭时等）。
4. **以上都不行：** 标记为 Q3 陷阱。问："这件事感觉紧急但不重要——我们怎么走到这一步的？能避免这个模式吗？"

---

### Q4：不重要 + 不紧急 → 放弃
不重要也没截止日期。这些是伪装成任务的干扰。

**信号：** 没有截止日期、没有实质影响、"有了不错"但没人会想念它、别人的优先级但不符合你的目标。

**行动：** 建议放弃。直说："建议放弃这个——不重要也不紧急，不值得占用认知空间。"

---

## 如何分类

新事项出现时，评估两个维度：

**重要性** — 这件事能否实质性推进你的某个工作流或人生目标？用工作流优先级排序作为参考，同时考虑：不做会后悔吗？能预防更大的问题吗？

**紧急性** — 有真正的截止日期吗？"真正的"指外部的、有后果的（提交日期、有人在等、即将过期的机会），不是自己设定的或随意定的。

拿不准时，倾向于归入 Q2（重要但不紧急）而非 Q1。大多数事情感觉比实际更紧急。

---

## 回复格式

分类新事项时，使用此格式：

```
新事项：[简述]
象限：Q[1-4] — [重要/不重要] + [紧急/不紧急]
原因：[一句话解释分类理由]
建议行动：[怎么处理——安排、委派、放弃等]
优先级：P[1-3]（如果加入周目标）

不同意？告诉我更多信息，我会重新分类。
```

---

## 重新分类

如果有额外信息改变了分类（比如"其实明天就到期了"或"这个不重要，跳过"）：
1. 确认新信息
2. 用相同格式展示更新后的分类
3. 相应更新工作流.md 和/或周目标.md

---

## 周回顾整合

周计划时，回顾本周出现的 Q3 事项：
- 处理效率如何？
- 有没有规律？（比如同类紧急但不重要的事反复出现）
- 能否建立系统来预防或自动处理？

同时检查 Q2 事项——有没有被忽视的风险？标记出来。
