# 认知层级

## L1 — 深度投入
设计、头脑风暴、架构调试、战略决策。需要全部注意力。
- 半天内最好只做一个 L1 领域的工作（灵活掌握）
- 最适合安排在深度工作时段
- 不能并行处理

## L1.5 — 范围界定 / 任务分派
把决策转化为具体的执行规格和任务拆解。
- 启动 L2 工作之前必须先做这一步
- 界定不清 → agent 产出质量差 → 又要回到 L1 重来
- 花 15-30 分钟界定清楚，能省下好几个小时

## L2 — 轻度参与（监督执行）
Agent 在执行，你在审核。可能升级为 L1。
- **绝不同时做两个 L2** — 这是认知切换最大的坑
- 运动、做饭时可以做（节奏慢一点）
- 一个 L2 + 任意数量 L3 是最佳搭配

## L3 — 极轻度（自主运行）
部署、批量生成、日常任务。等完成就好。
- 任何时候都可以并行，包括带娃时
- 极小概率升级为 L2/L1
- 想开多少个就开多少个
